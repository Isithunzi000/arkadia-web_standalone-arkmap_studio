W przeglądarce: koniecznie wpisz listę zmian (szablon PR wymaga). GitHub Action automatycznie doda wizualny diff mapy jako komentarz. Po merge do mastera — Delwing skasuje branch "development".
✕
Branch już istnieje
Branch "development" ma oczekujące zmiany

Branch development już istnieje w repozytorium — są oczekujące zmiany do zmergowania. Nie można utworzyć nowego brancha o tej samej nazwie.

✓ Pobranie SHA refs/heads/master
✕ Tworzenie brancha "development" — już istnieje
Skontaktuj się z Delwingiem — musi zaakceptować lub odrzucić poprzednie PR i skasować branch "development". Potem spróbuj ponownie.
✕
