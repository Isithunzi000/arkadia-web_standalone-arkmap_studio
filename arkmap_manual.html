1. Wprowadzenie

ArkMap Studio to przeglądarka i edytor map świata Arkadia MUD działający w przeglądarce internetowej. Aplikacja nie wymaga instalacji — wystarczy otworzyć plik HTML. Cały kod, style i logika zawarte są w jednym pliku.

ArkMap Studio obsługuje dwa formaty plików:

  • .arkmap — autorski format JSON projektu ArkMap Studio (pełna wierność danych, rozszerzalne metadane)
  • .dat — binarny format Mudlet (natywny format map Mudleta, kompatybilny z klientem gry)

Możesz wczytać mapę z pliku lokalnego lub pobrać aktualną wersję bezpośrednio z repozytorium GitHub (Delwing/arkadia-mapa).

2. Szybki start

  1. Otwórz plik arkmap_editor_v1-4-22.html w przeglądarce (Chrome, Firefox, Edge)
  2. Kliknij 🌐 Wczytaj mapę online… aby pobrać aktualną mapę Arkadii z GitHuba, lub upuść plik .arkmap / .dat na strefę drop w sidebarze
  3. Wybierz obszar z listy po lewej stronie — mapa się wyświetli
  4. Nawiguj: przeciągaj mapę myszą, scrolluj aby zoomować, klikaj pokoje aby zobaczyć szczegóły
  5. Naciśnij Ctrl+E aby włączyć tryb edycji

4. Wczytywanie mapy

Metoda 1: Drag & drop

Przeciągnij plik .arkmap lub .dat na strefę „Upuść plik" w sidebarze. Format zostanie rozpoznany automatycznie.

Metoda 2: Przyciski

  • Wczytaj .arkmap… — otwiera okno wyboru pliku dla formatu .arkmap (JSON)
  • Wczytaj Mudlet .dat — otwiera okno wyboru pliku dla binarnego formatu Mudlet

Metoda 3: Wczytaj mapę online

Przycisk 🌐 Wczytaj mapę online… pobiera aktualną wersję mapy z repozytorium Delwing/arkadia-mapa na GitHubie. Wymaga połączenia z internetem. Przed pobraniem wyświetlany jest dialog potwierdzenia z informacją o źródle i rozmiarze pliku.

Przy wczytywaniu pliku .arkmap uruchamiana jest walidacja (sprawdzenie struktury JSON) i weryfikacja checksumów CRC-32. Jeśli wykryte zostaną problemy, zobaczysz dialog z listą błędów i wyborem: „Załaduj mimo to" lub „Anuluj".

Konwersja

Wczytanie pliku .dat automatycznie konwertuje go do wewnętrznego formatu .arkmap. Żadne dane nie są tracone — konwersja jest bezstratna i odwracalna. Obsługiwane wersje formatu Mudlet: 17–22. Zapis odbywa się zawsze w wersji 20 (standard Arkadii).

6. Sidebar — panel boczny

Nagłówek

Wyświetla nazwę projektu, wersję edytora/viewera, autorów. Kliknięcie na napis „ARKMAP Studio" otwiera dialog O programie z informacjami o licencji, linkach i wersji.

Sekcja Wczytaj

Strefa drag&drop oraz trzy przyciski wczytywania opisane w rozdziale Wczytywanie mapy.

Sekcja Zapisz

Przyciski zapisu (aktywne po wczytaniu mapy):

  • Zapisz .arkmap / Zapisz .arkmap… — eksportuje mapę do formatu JSON .arkmap (po pierwszym zapisie przez File System Access API tekst zmienia się na „Zapisz .arkmap" bez „…")
  • Zapisz jako… — widoczny tylko w trybie edycji; zapisuje do nowego pliku
  • Eksportuj Mudlet .dat — konwertuje mapę z powrotem do binarnego formatu Mudlet

Wyszukiwarka

Pole tekstowe „Szukaj (pokój / region / id)…" — wyszukuje po nazwie pokoju, nazwie obszaru lub numerze ID. Więcej w rozdziale Wyszukiwarka.

Lista obszarów

Lista wszystkich obszarów (regionów) mapy. Kliknięcie wybiera obszar i wyświetla go na mapie. Aktualnie wybrany obszar podświetlony jest niebieską belką po lewej stronie.

Sortowanie

Przycisk ⇅ obok napisu „Obszary" otwiera dropdown z opcjami sortowania:

  • Nazwa A → Z / Z → A
  • Numer rosnąco / malejąco

Legenda

Zwinięty panel na dole sidebara — kliknij „Legenda Arkadii" aby rozwinąć. Więcej w Legenda Arkadii.

7. Panel pokoju (podgląd)

Po kliknięciu na pokój pojawia się floating panel w prawym górnym rogu ekranu. Zawiera:

  • Nagłówek — nazwa i ID pokoju, przycisk blokady pozycji (🔓 LOCK), przycisk zwijania (▲), przycisk zamknięcia (✕)
  • Dane pokoju — ID, pozycja (x, y, z), środowisko (kolor + nazwa), symbol (jeśli jest), waga (jeśli ≠ 1), hash (jeśli jest)
  • Wyjścia — lista kierunków (n, ne, e, se, s, sw, w, nw, up, down, in, out) z numerem docelowego pokoju; kliknij numer aby przeskoczyć do pokoju
  • Stubs — nieznane wyjścia (kierunki bez celu), jeśli są
  • Drzwi, wagi wyjść, zablokowane wyjścia — jeśli są
  • Wyjścia specjalne — komendy i ich cele
  • Custom lines — rysowane odcinki z kolorem i stylem

Blokada pozycji panelu

Kliknij 🔓 LOCK aby zablokować panel w miejscu. Po zablokowaniu (🔒) panel nie zmienia pozycji nawet gdy klikniesz na inny pokój. Kliknij ponownie aby odblokować.

Przeciąganie i zmiana rozmiaru

Panel można przeciągnąć chwytając za nagłówek. Rozmiar zmienisz przeciągając prawy dolny róg panelu.

Przycisk „skocz do pokoju" (⊙)

Centruje widok mapy na zaznaczonym pokoju.

8. Ustawienia

Kliknij przycisk ⚙ (lewy górny róg canvasa) aby otworzyć panel ustawień. Dostępne opcje:

OpcjaDomyślnieOpis
Papier milimetrowyONWyświetla subtelną siatkę w tle canvasa — pomaga wizualnie ocenić odległości
MinimapaONWłącza/wyłącza miniaturowy podgląd obszaru w prawym dolnym rogu
Pokaż suppressoryONWyświetla symbol ⊘ przy krawędziach pokojów, które mają exit suppressory (zaawansowane)
Oryginalne pixmapyOFFWyświetla etykiety obrazkowe (pixmap) z oryginalnego pliku Mudlet zamiast tekstu
Wszystkie ustawienia są zapamiętywane w localStorage przeglądarki. Przycisk ↺ Przywróć ustawienia domyślne na dole panelu resetuje pozycje i rozmiary okien, ustawienia wyświetlania, preferencje edytora — czyści cały localStorage ArkMap.

Sekcja Edytor

Ustawienia związane z trybem edycji, w tym domyślny czas locka dla sesji GitHub (obecnie nieaktywne — wymaga akceptacji Delwinga).

Sekcja Konto GitHub

Status połączenia z GitHubem — zalogowany/niezalogowany, przycisk łączenia konta, przycisk wylogowania.

9. Minimapa

Miniaturowy podgląd całego aktualnego obszaru wyświetlany w prawym dolnym rogu. Biała ramka oznacza aktualny viewport (to co widzisz na canvasie). Kliknij na minimapie aby szybko przeskoczyć do innej części obszaru.

Minimapę można włączyć/wyłączyć w Ustawieniach.

10. Wyszukiwarka

Pole wyszukiwania znajduje się w sidebarze między przyciskami wczytywania a listą obszarów. Aktywuje się po wczytaniu mapy. Skrót: Ctrl+F ustawia focus na polu.

Wpisz fragment nazwy pokoju, nazwę obszaru lub numer ID. Wyniki pojawiają się na liście pod polem — kliknij wynik aby przejść do pokoju na mapie. Wyszukiwanie przeszukuje wszystkie obszary jednocześnie.

11. Planer trasy

Planer trasy to zaawansowane narzędzie do planowania ścieżek między pokojami. Otwórz go klikając przycisk z ikoną trasy na lewej krawędzi canvasa (pod przyciskiem ⚙).

Jak używać

  1. Otwórz planer (przycisk z ikoną trasy)
  2. Kliknij + Dodaj waypoint — pojawi się nowy punkt na liście
  3. Kliknij pokój na mapie lub wpisz nazwę/ID — aktywny waypoint (oznaczony ►) przyjmie cel
  4. Dodaj kolejne waypointy — trasa obliczy się automatycznie

Tryb planowania

Kliknij Planuj w nagłówku panelu — włącza tryb, w którym każde kliknięcie na pokój na mapie automatycznie dodaje waypoint. Na ekranie pojawi się pulsujący wskaźnik „✏ tryb planowania".

Algorytmy pathfindingu

AlgorytmOpis
DijkstraKlasyczny algorytm najkrótszej ścieżki — zawsze znajduje optymalną trasę, gwarantowane minimum
A*Zoptymalizowana wersja z heurystyką — szybszy przy dużych mapach, identyczny wynik. Automatycznie przełącza się na Dijkstrę dla tras cross-area (różne układy współrzędnych).
Pathfinding uwzględnia: wagi pokojów (weight), wagi wyjść (exit_weights), wyjścia specjalne z wagami. Pokoje z flagą locked są pomijane (nie można przez nie przejść, ale można do nich dotrzeć jako cel). Zablokowane wyjścia (exit_locks, special_exit_locks) są ignorowane. Drzwi (doors) nie blokują pathfindingu.

Prędkość

Ustaw prędkość w sekundach na pokój (domyślnie 3s). Presety: 1s, 3s, 5s, 10s. Czas podróży wyświetlany jest w podsumowaniu trasy.

Podsumowanie trasy

Na dole panelu widać: łączną liczbę pokojów, łączną wagę trasy, szacowany czas podróży, informacje o drzwiach na trasie.

Kod trasy

Pole Kod trasy wyświetla serializowaną trasę w formacie ARKMAP:.... Możesz skopiować ten kod (przycisk ⎘) i udostępnić innej osobie, która zaimportuje go w swoim ArkMap Studio.

Import trasy

Kliknij Importuj w nagłówku panelu → wklej kod ARKMAP:... w oknie dialogowym → kliknij „Importuj trasę".

Dodatkowe akcje

  • ⊙ Pokaż trasę — centruje widok mapy na obliczonej trasie
  • ⇅ Odwróć trasę — odwraca kierunek (zamień start i koniec)
  • ⊗ Wyczyść — usuwa wszystkie waypointy

Miniaturowy podgląd trasy

W lewym dolnym rogu canvasa (gdy planer jest otwarty) pojawia się mały podgląd trasy z zaznaczonymi waypointami na tle obszaru.

12. Eksport widoku (PNG / SVG)

Kliknij przycisk eksportu (ikona strzałki w dół, na lewej krawędzi canvasa pod przyciskiem planera). Pojawi się panel z opcjami:

  • Format — PNG (rastrowy) lub SVG (kontener SVG z osadzonymi obrazami PNG — mapa i panele HUD jako raster, format pliku .svg)
  • Z elementami HUD — włącza/wyłącza widoczność floating paneli (panel pokoju, planer) w eksporcie

Dwa przyciski akcji:

  • ↓ Pobierz — zapisuje obraz do pliku
  • ⎘ Kopiuj — kopiuje obraz do schowka

13. Legenda Arkadii

Rozwijany panel na dole sidebara. Zawiera trzy sekcje:

Środowiska

Lista wszystkich kolorów środowisk używanych w Arkadii (las, trakt, miasto, wieś itd.) z kolorowym próbnikiem i numerem envId. Kliknięcie na środowisko filtruje widok.

Symbole

Lista symboli pokojów (P = Poczta, K = Kowal, S = Sklep, T = Karczma itd.).

Połączenia

Wizualna legenda typów połączeń między pokojami:

  • Linia ciągła — normalne wyjście
  • Linia z ◇ — drzwi otwarte
  • Linia z ◆ — drzwi zamknięte
  • Linia z ⊠ (pomarańczowa) — drzwi na klucz
  • Linia przerywana pomarańczowa — wyjście do innego obszaru
  • Linia przerywana żółta — wyjście specjalne (komenda)
  • Linia czerwono-zielona — custom line (rysowana ręcznie)

14. Tryb edycji EDYTOR

Tryb edycji pozwala modyfikować mapę — dodawać, usuwać i edytować pokoje, wyjścia, obszary, etykiety i custom lines.

Włączanie trybu edycji

Naciśnij Ctrl+E lub kliknij przycisk ✎ Edycja w pasku statusu. Pojawi się dialog wyboru trybu:

  • 💾 Edycja lokalna — edycja offline, zapis do .arkmap lub .dat w dowolnym momencie
  • ⬡ Sesja GitHub — (wkrótce) edycja z synchronizacją z repozytorium GitHub, lock, Pull Request

Sygnalizacja trybu edycji

Gdy tryb edycji jest aktywny:

  • Ramka sidebara zmienia kolor na pomarańczowy
  • Pasek statusu podświetla się na ciepły odcień
  • W pasku statusu pojawia się pulsująca kropka „EDYCJA LOKALNA"
  • Panel pokoju zmienia kolor akcentów na pomarańczowy
  • Pojawia się toolbar edycji (⊕ Dodaj pokój, 🎨 Narzędzia, ⚑ Sprawdź spójność)

🎨 Narzędzia rysowania

Przycisk 🎨 Narzędzia w toolbarze otwiera floating panel z dwiema zakładkami:

  • Kolory — paleta środowisk. Kliknij kolor aby ustawić środowisko zaznaczonego pokoju. Prawy klik na kolorze otwiera dialog edycji koloru środowiska (zmiana RGB dotyczy wszystkich pokoi z tym envId).
  • Symbole — paleta symboli. Kliknij symbol aby ustawić na zaznaczonym pokoju. Przycisk „✕ Wyczyść symbol" usuwa symbol.

Undo / Redo

Każda zmiana jest zapisywana na stosie historii (limit: 50 operacji). Cofaj (Ctrl+Z) i ponawiaj (Ctrl+Y). W pasku statusu dostępne są cztery przyciski: ↩↩ (cofnij wszystko), ↩ (cofnij), ↪ (ponów), ↪↪ (ponów wszystko). System obsługuje 24 typy operacji — od edycji pokojów i wyjść, przez etykiety i custom lines, po usuwanie obszarów i automatyczną naprawę suppressorów.

Licznik zmian

W pasku statusu widoczny jest licznik zmian (np. „● 12"). Kliknij go aby otworzyć Historię zmian (sesja) — chronologiczną listę operacji w bieżącej sesji edycji.

Wyjście z trybu edycji

Naciśnij Ctrl+E ponownie. Jeśli masz niezapisane zmiany, pojawi się dialog z opcjami: Porzuć / Zapisz .arkmap.

15. Edycja pokoju EDYTOR

W trybie edycji panel pokoju przełącza się na widok edycji z sześcioma zakładkami:

Zakładka: Podstawowe

  • Nazwa — nazwa pokoju (nie może być pusta)
  • Środowisko — dropdown z listą środowisk (envId)
  • Symbol — max 2 znaki (np. „P" dla Poczty, „[]" dla Tablicy)
  • Waga — koszt wejścia do pokoju dla pathfindingu (≥ 1, domyślnie 1)
  • Pozycja — X, Y, Z (edytowalne — zmiana wartości natychmiast przesuwa pokój na mapie z podglądem na żywo; możesz też przesuwać strzałkami lub dragiem)

Zakładka: Wyjścia

Interaktywna siatka 3×3 reprezentująca 8 kierunków + UP / DOWN / IN / OUT:

  • Kolor niebieski = wyjście istnieje
  • Kolor ciemny = brak wyjścia (kliknij aby dodać)
  • Kolor żółty przerywany = oczekuje na cel (tryb dodawania)
  • Kolor niebieski przerywany = stub (nieznane wyjście)

Po kliknięciu na istniejące wyjście pojawia się panel szczegółów:

  • Target ID — numer pokoju docelowego
  • Drzwi — brak / open / closed / locked
  • Waga wyjścia — dodatkowy koszt krawędzi
  • Stub — oznacz jako stub (wyjście bez celu) lub odznacz aby usunąć istniejący stub
  • Exit lock — zablokowane wyjście
Aby usunąć istniejący stub: kliknij na komórkę z kierunkiem stuba, odznacz checkbox Stub w panelu szczegółów, a następnie zapisz pokój. Stub zostanie usunięty.

Dodawanie wyjścia z panelu szczegółów

Kliknij na pustą komórkę (ciemną) w siatce wyjść, wpisz ID pokoju docelowego w polu Target ID i kliknij przycisk + Dodaj wyjście. Jeśli pokój docelowy nie ma jeszcze wyjścia w kierunku powrotnym, pojawi się dialog z propozycją dodania wyjścia powrotnego (dwukierunkowego). Możesz wybrać „Dodaj obustronnie" lub „Tylko w jedną stronę".

Dialog wyjścia powrotnego pojawia się też po zapisie pokoju (✓ Zapisz pokój), jeśli w trakcie edycji dodano nowe wyjścia do pokojów, które nie mają jeszcze wyjścia w przeciwnym kierunku.

Zakładka: Specjalne

Wyjścia specjalne — komendy tekstowe (np. „wejdz do jaskini") prowadzące do innego pokoju. Każde wyjście specjalne ma komendę i Target ID.

Zmiana nazwy komendy: Przy zmianie tekstu komendy (np. „portal" → „brama") powiązane metadane (drzwi, waga, custom line) są automatycznie migrowane na nowy klucz, pod warunkiem że cel (Target ID) nie zmienił się. Jeśli zmienisz jednocześnie komendę i cel, metadane zostaną usunięte.
Usuwanie wyjścia specjalnego: Po usunięciu komendy z listy i zapisaniu pokoju, wszystkie powiązane metadane (drzwi, waga, custom line) są automatycznie czyszczone.

Zakładka: CL (Custom Lines)

Lista custom lines (ręcznie rysowanych połączeń) przypisanych do pokoju. Każda linia ma kierunek, kolor, styl (solid/dash/dot) i tablicę punktów. Na dole zakładki dostępne są dwa przyciski:

  • + Dodaj custom line… — otwiera dialog wyboru kierunku wyjścia, dla którego chcesz narysować nową linię
  • ⊘ Dodaj suppressor… — otwiera dialog wyboru kierunku, w którym chcesz ukryć domyślną linię (potrzebny gdy po drugiej stronie jest custom line)

Więcej o rysowaniu i suppressorach w Custom Lines i Suppressory.

Zakładka: Dane

  • Notatki .arkmap only — dowolny tekst przypisany do pokoju (nie zapisywany do .dat)
  • Tagi .arkmap only — etykiety/tagi (nie zapisywane do .dat)
  • User data — pary klucz-wartość (zachowywane w obu formatach)

Zakładka: Paleta

Szybki dostęp do palet środowisk i symboli. Kliknij na kolor aby natychmiast ustawić środowisko pokoju. Kliknij na symbol aby ustawić symbol (⬡ ustawia też odpowiedni kolor).

Przyciski akcji

  • ✓ Zapisz pokój (lub Enter) — zatwierdza zmiany
  • Usuń pokój — otwiera dialog potwierdzenia (informuje o wyjściach i referencjach)
  • Anuluj (lub Esc) — odrzuca niezapisane zmiany

Dodawanie nowego pokoju

Dwa sposoby:

  • Kliknij ⊕ Dodaj pokój w toolbarze, potem kliknij na mapę
  • Shift+Klik na pustym miejscu mapy

Przesuwanie pokoju

  • Zaznacz pokój i użyj ←↑↓→ (przesuwa o 1)
  • Shift+←↑↓→ — przesuwa o 5 jednostek
  • Drag pokoju myszą na canvasie

Nawigacja między pokojami

Tab — przeskakuje do następnego pokoju na bieżącym poziomie Z.

16. Edycja obszaru EDYTOR

W trybie edycji panel obszaru otwiera się po dwukrotnym kliknięciu na nagłówek obszaru w sidebarze lub przez menu kontekstowe.

Panel obszaru zawiera:

  • Nazwa obszaru — edytowalna, musi być unikalna
  • Notatki obszaru — opis obszaru
  • Etykiety na mapie — lista tekstowych i obrazkowych etykiet z kontrolami: edycja (✎), usunięcie (✕), dodawanie nowej
  • user_data — pary klucz-wartość na poziomie obszaru
  • Informacje — ID, liczba pokoi, is_zone

Dodawanie nowego obszaru

Przycisk + Nowy obszar pod listą obszarów otwiera dialog. Nazwa musi być unikalna.

Usuwanie obszaru

Przycisk ✕ Usuń w panelu obszaru. Jeśli obszar zawiera pokoje, wyświetlone zostanie ostrzeżenie.

Edycja etykiet

Etykiety to tekstowe napisy lub obrazki (pixmap) na mapie. Dialog edycji etykiety pozwala zmienić: tekst, pozycję (X, Y, Z), rozmiar, kolory tekstu i tła, opcje skalowania i warstwy.

Przesuwanie i skalowanie etykiet

Zaznaczoną etykietę można przesuwać (drag) chwytając za jej ciało oraz skalować ciągnąc za narożniki. Każde przesunięcie i zmiana rozmiaru tworzy wpis w historii undo/redo. Jeśli w trakcie przeciągania lub skalowania naciśniesz Esc, operacja zostanie anulowana — etykieta wraca do oryginalnej pozycji i rozmiaru, a wpis nie trafia do historii.

Nowa etykieta z panelu obszaru

Etykiety można dodawać z menu kontekstowego (prawym klikiem na pustym miejscu mapy → „Dodaj etykietę tutaj") lub z panelu obszaru (przycisk „⊕ Dodaj etykietę"). W drugim przypadku nowa etykieta pojawi się w centrum widocznego viewportu.

17. Custom Lines i Suppressory

Custom Lines

Custom lines to ręcznie rysowane linie na mapie zastępujące domyślne proste połączenia między pokojami. Służą do wizualnego przedstawienia krętych dróg, rzek, tuneli itp.

Każda custom line składa się z:

  • Kierunek — powiązana z wyjściem (n, e, s itd.) lub wyjściem specjalnym
  • Punkty — tablica współrzędnych [x, y] definiujących przebieg linii
  • Kolor — RGB
  • Styl — solid, dash, dot, dash_dot, dash_dot_dot
  • Strzałka — czy zakończona strzałką

Rysowanie custom line

  1. W trybie edycji otwórz zakładkę CL w panelu pokoju
  2. Kliknij „Rysuj" przy wybranym wyjściu (lub użyj „+ Dodaj custom line…" aby wybrać kierunek)
  3. Kliknij na mapie aby dodawać kolejne punkty linii
  4. Zakończ rysowanie: podwójny klik, Enter lub prawy klik
  5. Esc anuluje rysowanie

Exit Suppressory

Suppressor to custom line z pustą tablicą punktów ("points": []). Jego jedynym zadaniem jest ukrycie domyślnej linii wyjścia po stronie docelowego pokoju. Jest to celowy mechanizm Mudleta — nie błąd danych.

Ważne: jeśli pokój A ma custom line do pokoju B w kierunku E, to pokój B powinien mieć suppressor w kierunku W. Bez tego Mudlet narysuje obie linie — custom i domyślną — co wygląda źle.

Sprawdzanie spójności

Przycisk ⚑ Sprawdź spójność w toolbarze edytora skanuje wszystkie pokoje i wykrywa brakujące suppressory. Wynik: dialog z listą problemów i przyciskiem „Napraw automatycznie".

Przy eksporcie do .dat (jeśli jesteś w trybie edycji) sprawdzenie uruchamia się automatycznie — masz opcje: „Eksportuj mimo to", „Napraw i eksportuj" lub „Anuluj eksport".

19. Zapisywanie mapy

Zapisz .arkmap

Skrót: Ctrl+S. Eksportuje pełną mapę do formatu .arkmap (JSON). Przy pierwszym zapisie pojawia się dialog wyboru pliku. Przy kolejnych zapisach (jeśli przeglądarka obsługuje File System Access API) zapis odbywa się do tego samego pliku bez dialogu.

W trybie edycji dostępny jest dodatkowy przycisk Zapisz jako… — pozwala zapisać mapę do nowego pliku niezależnie od ustawionego uchwytu File System Access.

Przed zapisem obliczane są hierarchiczne checksums CRC-32 (pokoje → obszary → plik). Tablice sortowane są deterministycznie (ID rosnąco) aby plik był przyjazny dla systemów kontroli wersji (git diff).

Eksportuj Mudlet .dat

Konwertuje mapę z formatu .arkmap do binarnego formatu Mudlet i pobiera plik. Jeśli tryb edycji jest aktywny, przed eksportem uruchamiana jest automatyczna kontrola suppressorów.

Przy eksporcie do .dat niektóre dane specyficzne dla .arkmap (notatki, tagi) nie zostaną zapisane — format Mudlet ich nie obsługuje.

23. Obsługiwane formaty plików

.arkmap

Autorski format projektu ArkMap Studio. Plik JSON z deterministyczną serializacją (posortowane klucze, inline arrays prymitywów). Zawiera:

  • Pełne dane mapy (pokoje, wyjścia, drzwi, wagi, custom lines, etykiety, obszary)
  • Metadane (nazwa mapy, font symboli, user_data)
  • Palety kolorów (env_colors, custom_env_colors)
  • Rozszerzenia .arkmap-only: notatki, tagi
  • Hierarchiczne checksums CRC-32 do weryfikacji integralności

Pełna specyfikacja formatu: patrz osobny dokument ArkMap Format Specification.

.dat (Mudlet)

Binarny format map klienta Mudlet (QDataStream). Odczyt obsługuje wersje 17–22. Zapis zawsze w wersji 20 (standard Arkadii). Konwersja do/z .arkmap jest bezstratna — roundtrip .dat → .arkmap → .dat daje byte-identyczny wynik (potwierdzone testami SHA-256).

24. FAQ — pytania i odpowiedzi

Czy muszę cokolwiek instalować?

Nie. Otwórz plik HTML w dowolnej nowoczesnej przeglądarce. Cały kod jest w jednym pliku.

Czy mogę edytować mapę offline?

Tak. Tryb „Edycja lokalna" działa bez internetu. Potrzebujesz internetu tylko do: wczytania mapy online, logowania GitHub, wysyłania Pull Requestów.

Co jeśli checksum nie zgadza się przy wczytywaniu?

To ostrzeżenie, nie blokada. Oznacza, że plik został zmodyfikowany poza ArkMap Studio (np. ręcznie edytowany JSON). Możesz go wczytać klikając „Załaduj mimo to".

Co to jest suppressor?

Custom line z pustą tablicą punktów. Ukrywa domyślną linię wyjścia rysowaną przez Mudlet. Jest potrzebny gdy po drugiej stronie jest custom line — bez suppressora rysowane byłyby dwie linie.

Czy dane z zakładki Dane (notatki, tagi) zapiszą się do .dat?

Nie. Notatki i tagi to rozszerzenia formatu .arkmap. Przy eksporcie do .dat zostaną pominięte. user_data zapisuje się w obu formatach.

Jak wrócić po zgubieniu się na mapie?

Naciśnij F (Fit to view) — widok dopasuje się do granic aktualnego obszaru.

Czy mogę pracować na tym samym pliku co inna osoba?

W trybie lokalnym — nie ma mechanizmu blokowania. W przyszłości sesja GitHub zapewni locking (rezerwację). Używaj kontroli wersji (git) do współpracy.

Jakie przeglądarki są wspierane?

Chrome, Firefox, Edge (najnowsze wersje). File System Access API (automatyczny zapis bez dialogu) działa tylko w Chrome/Edge.
